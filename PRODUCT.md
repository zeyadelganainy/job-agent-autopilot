# Product

## Register

product

## Users
A single job-seeker (the owner) running JobPilot as an always-on personal agent. They
check in from a laptop, occasionally a phone, to review what the agent prepared overnight
and decide what to apply to. A secondary audience is **recruiters**, who enter the
no-live-calls demo from the login page to evaluate the product end-to-end.

The user is in a *triage + review* workflow: scan the day's matches, skim prepared
résumé/cover-letter sets, apply manually, and track outcomes. Speed and trust matter more
than discovery — they want to see the agent's work and act on it without friction.

## Product Purpose
An autonomous job-application **prep** agent: scan ATS feeds → score against the user's
profile → auto-generate tailored documents → surface them in a monitoring console for the
human to review and apply. The console is the cockpit: dashboard (today's stats, ready-to-
apply queue, triage, run history), jobs board, ad-hoc generate, docs library, application
tracker, insights, settings. Success = the user trusts the queue and applies in minutes.

## Brand Personality
Sharp, capable, quietly premium — a focused instrument, not a toy. Three words:
**precise, confident, calm.** It should feel like a modern developer tool (Linear / Vercel /
Raycast lineage): dense where it needs to be, legible, fast, with restraint over decoration.

## Anti-references
- The previous look: flat light-blue "Swiss SaaS," generic and forgettable.
- Bootstrap-era admin dashboards; gradient hero-metric cards; rainbow chart palettes.
- Consumer-cute or playful styling. This is a tool, not a marketing page.

## Design Principles
1. **The tool disappears into the task.** Earned familiarity over novelty; standard
   affordances, consistent vocabulary screen to screen.
2. **Signal over chrome.** The match-score and run state are the heroes; accent color marks
   action and status only, never decoration.
3. **Trustworthy density.** Tables and panels can be dense, but never cramped or noisy —
   rhythm and hierarchy carry the eye.
4. **Calm confidence.** Motion conveys state (150–250ms), never performs. No load
   choreography; the cockpit is just *there*.

## Accessibility & Inclusion
Body text ≥4.5:1 against its surface; focus-visible rings on every interactive element;
full `prefers-reduced-motion` fallback. Status is never color-only (paired with text/shape).
Keyboard-operable dialogs (Esc/Enter, focus management).
